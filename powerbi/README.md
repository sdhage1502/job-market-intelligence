# Power BI Analytics Dashboard Specifications

This directory contains specifications and configuration guides to wire the Star Schema database (`dim_company`, `dim_location`, `dim_skill`, `dim_date`, `dim_salary`, `fact_jobs`) into Power BI Desktop to build the 5 requested analytics dashboard pages.

---

## Data Model Connections & Relationships

In Power BI **Model View**, set up relationships between tables using the surrogate keys generated by the dbt pipeline:

```text
               ┌───────────────────────┐
               │      dim_company      │
               │   [company_key] (1)   │
               └───────────┬───────────┘
                           │ (1:N)
               ┌───────────▼───────────┐
               │       fact_jobs       │
               │  [location_key] (N)   │
               └───────────▲───────────┘
                           │ (N:1)
               ┌───────────┴───────────┐
               │     dim_location      │
               │   [location_key] (1)  │
               └───────────────────────┘
```

Complete Relationship Matrix:
* `fact_jobs[company_key]` (Many) ➔ `dim_company[company_key]` (One) - Active, Single filter direction.
* `fact_jobs[location_key]` (Many) ➔ `dim_location[location_key]` (One) - Active, Single filter direction.
* `fact_jobs[date_key]` (Many) ➔ `dim_date[date_key]` (One) - Active, Single filter direction.
* `fact_jobs[salary_key]` (Many) ➔ `dim_salary[salary_key]` (One) - Active, Single filter direction.
* For skills analysis: Join `mart_skill_trends` view to `dim_date` on `posted_date` = `full_date` and filter by skill dimension fields.

## DAX Measures (Core Metrics)

Create a measure table `[# Measures]` in Power BI and paste the following DAX calculations:

### 1. Total Jobs
```DAX
Total Jobs = COUNTROWS('fact_jobs')
```

### 2. Average Salary
```DAX
Average Salary = AVERAGE('dim_salary'[avg_salary])
```

### 3. Remote Jobs %
```DAX
Remote Jobs % = 
DIVIDE(
    CALCULATE([Total Jobs], 'dim_location'[is_remote] = TRUE),
    [Total Jobs],
    0
)
```

### 4. Month-over-Month Job Growth
```DAX
MoM Job Growth % = 
VAR CurrentMonthJobs = [Total Jobs]
VAR PreviousMonthJobs = 
    CALCULATE(
        [Total Jobs],
        DATEADD('dim_date'[full_date], -1, MONTH)
    )
RETURN
    DIVIDE(CurrentMonthJobs - PreviousMonthJobs, PreviousMonthJobs, 0)
```

### 5. Skill Growth %
```DAX
Skill Growth % = 
VAR CurrentMonthCount = SUM('mart_skill_trends'[skill_frequency])
VAR PreviousMonthCount = 
    CALCULATE(
        SUM('mart_skill_trends'[skill_frequency]),
        DATEADD('dim_date'[full_date], -1, MONTH)
    )
RETURN
    DIVIDE(CurrentMonthCount - PreviousMonthCount, PreviousMonthCount, 0)
```

### 6. Median Salary
```DAX
Median Salary = MEDIAN('dim_salary'[avg_salary])
```

### 7. Highest Salary
```DAX
Highest Salary = MAX('dim_salary'[max_salary])
```

## Visual Layouts & Dashboard Pages

### Page 1: Executive Overview
* **Goal**: High-level telemetry of the current job market for executives.
* **Layout**:
  * **Top Row Cards (KPIs)**: `Total Jobs` | `Average Salary` | `Remote Jobs %` | `MoM Job Growth %`.
  * **Primary Chart (Line Chart)**: Job postings volume over time (X-axis: `dim_date[full_date]` monthly hierarchy, Y-axis: `Total Jobs`).
  * **Secondary Chart (Donut Chart)**: Distribution of jobs across major sectors/industries (`dim_company[industry]`).
  * **KPI Card**: Most in-demand skill (`dim_skill[skill_name]` filtered by top 1).

### Page 2: Skill Trends
* **Goal**: Drills down into the specific tools and programming languages hiring managers want.
* **Layout**:
  * **Top Skills (Clustered Bar Chart)**: Y-axis: `dim_skill[skill_name]`, X-axis: `Total Jobs` (filtered to top 15).
  * **Growth Heatmap (Matrix Visual)**: Rows: `dim_skill[skill_category]` / `skill_name`, Columns: `dim_date[month]`, Values: `Skill Growth %` (styled with conditional formatting color gradients).
  * **MoM Trend Line**: Tracks selection growth (X-axis: `Date`, Y-axis: `Skill Growth %`, Legend: `Skill Category`).

### Page 3: Salary Analytics
* **Goal**: Benchmark compensation across roles, tech stacks, and locations.
* **Layout**:
  * **Top KPIs**: `Average Salary` | `Median Salary` | `Highest Salary`.
  * **Salary by Role (Bar Chart)**: Y-axis: `job_title`, X-axis: `Average Salary` (sorted descending).
  * **Salary by City (Clustered Column Chart)**: X-axis: `dim_location[city]`, Y-axis: `Average Salary` split by `Remote` vs `On-site`.
  * **Salary by Company (Matrix Table)**: Company name, average salary, highest salary, and job volume count.

### Page 4: Location Analytics
* **Goal**: Geographic concentrations and remote work growth indicators.
* **Layout**:
  * **Geographic Distribution (Map Visual)**: Location: `dim_location[city]`, Size: `Total Jobs`.
  * **Hiring by City (Treemap)**: Grouping: `dim_location[city]`, Values: `Total Jobs`.
  * **On-Site vs Remote Split (Pie Chart)**: Legend: `dim_location[is_remote]`, Values: `Total Jobs`.

### Page 5: Experience Analytics
* **Goal**: Analyze talent requirements (entry, mid, or senior level).
* **Layout**:
  * **Experience Distribution (Column Chart)**: X-axis: Experience bracket grouping:
    * *0-2 years (Entry)*
    * *2-5 years (Junior)*
    * *5-8 years (Mid-Senior)*
    * *8+ years (Lead/Principal)*
    Y-axis: `Total Jobs`.
  * **Experience vs Salary (Scatter Plot)**: X-axis: Experience required, Y-axis: Average Salary, Details: Job Title.
